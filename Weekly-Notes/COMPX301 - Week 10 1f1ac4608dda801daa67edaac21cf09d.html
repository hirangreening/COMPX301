How can we find solutions to NP-Hard problems?

Subset Selection

How can we find good solutions to local search and optimization problems that are a form of subset selection?

Subset selection:
The selection of the subset from the total set can be represented by a bit pattern
of 0's and 1's which represent whether the item has been selected or not.

e.g.

[x1, x2, x3, x4, x5, x6, ...]
[ 1   0   1   1   0   1, ...]

So if we want to move an item from one subset to the other, we just flip a bit from
0 to 1 or vice versa. 

If two of these bit patterns have only one bit difference they are considered:
Neighbouring solutions

e.g. 
1 0 1 1 0 1

Would be neighbours with:

1 0 1 1 1 1
and 
1 0 1 0 0 1

Generate and test:

The “naive” way to find a solution.

  1. Repeatedly generate a candidate solution.
  1. Evaluate it.
  1. Always keep the best one.
  1. Stop when resources are exhausted (generally time).

Heuristic methods

Rule of thumb, educated guess.

These will likely be tested, but at a high level. You won’t be expected to know pseudo-code or anything. But as Tony said “you should probably be able to explain to your grandma how the genetic algorithm works”.

K-clique problem.

Find a subset of k vertices which is a complete graph, that is, a complete subgraph of size k.

for k=3 there are 336 combinations to check

for k=4 there are 1680 combinations to check

for k=5 there are 6730 combinations to check

… a factorial expansion of the search space.

A complete graph is where each node in the graph has a direct link to every other node.

Hill climbing:

  1. Choose a random starting set, S = {A, B, C, D, E}
  1. Evaluate S, say as the total number of adjacencies eval({A, B, C, D, E}) = 15
  1. Change one vertex in S to produce S’ = {A, B, C, D, F}
  1. If eval(S’)>eval(S) then S = S’
  1. If S is the local maximum then stop, else go to step 3.

Simulated annealing:

Try and make random progress toward the solution, gradually confining the search space.

t = initial temperature, how many changes allowed on each iteration.

r = rate of cooling, how quickly t diminishes

  1. Choose a random starting set,
  1. Evaluate S, total number of adjacencies
  1. Change as many vertices in S as allowed by t to produce S’
  1. if eval(S’) >eval(S) then S = S’
  1. cool t by r and if t > 0 go to 3.

Genetic algorithm:

Keep a population and gradually improve the fitness.

  1. Start with a population of solutions, G
  1. Evaluate fitness of each solution.
  1. Select subset of G to survive in G’
  1. Select two solutions in G’ for breeding
  1. Add offspring to G’
  1. Mutate weakest solution (optional)
  1. If more time allowed, go to step 2.

Assignment 4 - Tallest Box Stack.

Due: 7/6/2025

Description: 

Implement a simulated annealing solution to find the tallest stack possible for a given set of boxes, according to the specification given below. This assignment may be done in pairs or individually (n.b. students arrange their own partnerships). The solution must be written in Java to run on a Linux machine like those in the R Block labs.  (An option to try for a genetic algorithm approach instead requires explicit permission from the lecturer.)

Specification:

  • The dimensions of a box are specified with three integers—defining its height, width and depth (in no particular order)
  • A set of boxes is defined within a text file, where each line has three space-separated integers giving the dimensions of one box. (Several sample data files are supplied in the archive file below.)
  • A box can be oriented such that any one side is its bottom face. Thus, a box has at most three distinct orientations.
You may recall in lectures that Tony talked about creating an array and storing 
all boxes and all possible orientations in here (So it would be 3 * n in size where
n is the number of boxes). You may be able to reduce this number since the orientations
of a cube are all the same, for example.

With this assignment you would need to also add some kind of identifier to the box 
since each box can only be used once.
  • Boxes are placed one-on-top-of-another to make a stack.
  • Touching faces condition: one box may be placed on top of another provided its bottom face is smaller than the top face of the box upon which it rests. That is, its bottom face must fit entirely within the area of the top-face of the box underneath, without over-hang or flush-sides (i.e. strictly shorter in both dimensions).
For box a and box b
b can go ontop of a only if b.width < a.width and b.depth < a.depth
  • Single use condition: Each box can be used at most once in the final stack, in any orientation that allows it to satisfy the "touching faces condition". (N.b. dynamic programming will not solve this problem, but might be adapted to find an initial candidate solution.)
You can use the dynamic programming method, then remove all duplicates with the smallest
height to get a starting point, if you like.
  • Your program must seek to find the tallest stack possible for a given set of boxes without violating the "touching faces" or "single use" conditions (n.b. this likely means that you cannot use all the boxes, except with certain sets of boxes).
Your program doesn't have to find the optimal solution every time
  • The output of your program is a list of the boxes forming the tallest stack you can find that satisfies the conditions above. Each line of output specifies the dimensions of one box in the stack using three integers (separated by spaces), where the first integer is the width of the box, the second is its length and the third is its height, and these are followed by a fourth integer that states the total height of the stack at the point where this box is top-most. The lines of output should be in order from the top-most box down to the bottom-most.
output format:

Smallest
....
....
Largest 

width, length/depth, height, total height at current box.

e.g:

1 1 1 8
2 2 2 7
3 3 2 5
4 5 3 3
  • Your program must run from the command-line (i.e. as a console program) and take three arguments: the first is the name of a text file presumed to define a set of boxes as outlined above; the second is an integer specifying the initial temperature t, expressed as an integer greater than 0, that specifies the number of boxes that can be changed (inserted or removed or substituted) from the current candidate solution to derive a neighbouring solution, and the third is a real value r, where 0.1 <= r <= t and is the rate of cooling expressed as how much t is reduced after each new solution has been generated and tested (i.e. t = t - r). Some test input files are available on Moodle in the archive file below, but students are encouraged to create their own additional test data and share results with each other during experimentation.

Usage:

java NPCStack filename temperature coolingRate

java NPCStack boxes.txt 20 0.1


You may find some sources online that use something like:

t *= r

to calculate temperature reduction, but the outline specifies:

t = t - r

So that is what we will use for this assignment.
  • Each line of input from the file should be checked to ensure all box dimensions are positive integers. Any line of input with an error should just be ignored.
  • Your program class must be called NPCStack.java
  • Students must devise their own method for finding the initial candidate solution, and this should be described both in the source code documentation and in the README file.
