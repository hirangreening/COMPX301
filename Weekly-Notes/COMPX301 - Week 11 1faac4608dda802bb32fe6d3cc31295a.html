Compression

What do we want from compression?

  • Good compression: effectively reduces the number of bits required to represent data.
  • Adaptive: Adapts to the specific data that it is being used to compress
  • Lossless: Doesn’t lose any information when compressed.

Simple overview of the types of compression:

AdaptiveStatic
LosslessAdapts to the data and doesn’t lose any informationDoesn’t lose any information but uses a fixed encoding scheme that doesn’t adapt to the data
LossyAdapts to the data being compressed but loses some data upon compressionLoses information upon compression and uses fixed encoding that doesn’t adapt to the data being compressed.

LZ77

Encodes repeated patterns using triples. 

outputs triples:
<i, j, c>

i = Index back to start of repeated pattern (either from the pointer or the start).
j = Length of the pattern.
c = First mismatched symbol.

So if we have <2, 3, c>

we look back 2, copy 3, and then put a 'c' on the end.

LZ78

A modified version of LZ77 that only uses a double.

We put the ‘seen’ phrases into a dictionary and index into the dictionary to pull out previous phrases as opposed to indexing into previous parts of the text. This way we don’t need to have the length of the phrase as a variable, so the output can be reduced from triple to doubles.

Phrase ‘0’ is always considered an empty string.

<i, c> where 
i = is the phrase previous phrase number
c = first mismatched symbol after the previous phrase

aaabbabaabaaabab

Input:        a     aa    b     ba   baa   baaa  bab
phrase num:   1     2     3     4     5     6     7
Output:     (0,a) (1,a) (0,b) (3,a) (4,a) (5,a) (4,b)

The dictionary is structured as a trie where the 0th item is the root node.

Tree stucture for the above outputs

                    0
                a /  \ b
                1     3
             a /       \ a
              2         4
                     b / \ a
                      7   5
                           \ a
                            6

Tree structure for "abc"

     0
 a/ b|  \c
 1   2   3
 
  1     2     3
(0,a) (0,b) (0,c)

Decoding is performed using two arrays, used as a lexicon

This provides instant lookup for decoder.

Bit packing:

Since we know that the dictionary will start at 0, and no phrase number we see is going to be any more than 1 phrase higher than ones we’ve already seen before, we can efficiently pack phrase numbers.

We know, for instance, at index 1 there will only ever be the one empty phrase number, so we can just encode that with one bit. At index 3 there could be 3 phrase numbers so we must encode that with 2 bits. We still have to encode it with 2 bits even if the phrase number doesn’t require 2 bits (0 or 1) because the decoder doesn’t know what phrase numbers are coming up next.

So if we have “p” many phrases in our dictionary, that can be encoded minimally with the ceiling of log2(p).

p = phrase number
b = number of bits needed to encode

p, b
0, 1
1, 1
2, 2
3, 2
4, 3
5, 3
6, 3
7, 3
...

The number of phrases supported by each bit 
increment is double that supported by the previous bit

so for this sequence, assuming each character is 8 bits:

encoding:  (0,a) (1,a) (0,b) (3,a) (4,a) (5,a) (4,b)
bits used: (1,8) (1,8) (2,8) (2,8) (3,8) (3,8) (3,8)
